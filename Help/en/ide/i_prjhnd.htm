What's New in the MIDletPascal 3.x IDE \ Project Handling


Script Building Event Handling
since 3.0

Now you can handle several building events by batch script files located in the scripts subdirectory of the project root.

This might be useful for you for many reasons like running external tools your project may need, backup the generated JAR files, modify automatically JAD files, include more files to the JAR if some external condition occurs, or just copy automatically the JAR file to your cellphone after it is created successfully.

 Background Compilation
since 3.0

In MP 3.0 the building process takes place in the background, so you can continue editing your source files.

Just remember the IDE will take a snapshot of the source code files by the time you send the Build order to the IDE, so if you change a source file after that, that changes will not be included in the current compilation (you will need to rebuild to get those changes included).

 Current File Compilation
since 3.0

Now you can compile the current source file only if you want to. Just press Shift+F7 in the Code Editor, or click the menu item Project \ Compile Current File.

 Stop Build Process
since 3.0

Now you can stop the current build process by the menu item Project \ Stop Build Process, also accesible via the Ctrl+F2 key combination.
The building process will be stopped after the current building stage has finished. This means you will have to wait a little sometimes.
