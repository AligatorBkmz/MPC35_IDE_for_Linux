What's New in the MIDletPascal 3.x IDE \ Source Code Handling


Preprocessing
since 3.0

In the Code Editor's inner tab called Preprocess you will find an additional code viewer that will show you how the code will look for the compiler after the preprocessor takes care of it.

 Conditional Defines
since 3.0

Now you can define your own conditional defines by the $DEFINE directive, undefine them by the $UNDEF directive, and check them by the $IFDEF and $IFNDEF directives just like in any other well-known pascal dialect.

MP 3.0 defines by default the following coditional defines:

  • J2ME (platform related)
  • MPC (compiler related)
  • MIDLETPASCAL30 (ide related) -- note: MP 3.0 defines MIDLETPASCAL30, MP 3.1 defines MIDLETPASCAL30 and MIDLETPASCAL31, and so on

 History Keeping
since 3.0

In Settings \ Keep History menu item you will find the switch to enabled or disable this feature. If Keep History is on, the IDE will store source file histories in the history subdirectory in the project root.

In the Code Editor's inner tab called History, in it's Contents tab you will find a historical source file content viewer with two options: Send to Buffer (to overwrite the current code editor's content with that old version) and Rollback to this Version (to discard the newer versions and go back to that version of the file).

 Check Differences
since 3.0

In the Code Editor's inner tab called History, in it's Difference tab you will find the integrated difference tool that will let you see the differences between two versions of the file. The purple colored lines indicates modified ones, the red lines the removed ones, and the blue ones the added lines.

 Source Code Formatting
since 3.1

Now you can format your Pascal source code to meet the highest redability standards.
